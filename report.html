🚀 프로젝트 보고서: QR Dropzone

제작자: 유선재 (명지대학교 기계공학과 4학년)

1. 기획 배경 (Motivation)

팀 프로젝트나 발표 현장에서 내 PC의 파일을 타인의 모바일 기기로 전송해야 할 때, 카카오톡 로그인이나 클라우드 업로드는 개인정보 노출 및 번거로운 절차라는 진입장벽이 존재합니다. 본 프로젝트는 '로그인 없이, 케이블 없이, 단 1초 만에' 파일을 공유할 수 있는 직관적인 윈도우 유틸리티를 목표로 개발되었습니다.

2. 주요 기능 및 구성물 (Features)

  • 초경량 드래그 앤 드롭 UI: 항상 위(Always-on-top) 속성을 가진 반투명 드롭존 제공.
  • 실시간 로컬 IP 트래킹: 네트워크 환경 변화에 따라 전송용 URL 주소를 동적으로 생성.
  • 즉시 다운로드(Force Download): 스캔 시 브라우저 미리보기가 아닌 '파일 다운로드'를 즉시 실행하도록 HTTP 헤더 최적화.
  • 무설치 수신 환경: 수신자는 별도의 앱 설치 없이 기본 카메라 앱만으로 파일 수신 가능.

3. 실현 방법 및 기술 스택

Python 3.12 PyQt5 (GUI Framework) Socket & Threading HTTP Server Library

AI 도구 활용 과정: Gemini(AI)를 설계 파트너로 활용하여 네트워크 스레드와 GUI 스레드 간의 충돌을 방지하는 비동기 구조를 설계하였습니다. 특히 윈도우 환경에서의 파일 경로 처리 및 QR 코드 인식률 향상을 위한 데이터 인코딩 알고리즘 최적화 과정에서 AI의 피드백을 반영하였습니다.

[파일 드롭] ➔ [로컬 IP 확인] ➔ [HTTP 서버 개방] ➔ [QR 코드 생성] ➔ [모바일 수신]

4. 특장점 및 차별점 (Strengths)

  • 보안성(Privacy): 외부 클라우드 서버를 거치지 않고 로컬 네트워크 내에서 직접 전송하므로 데이터 유출 위험이 현저히 낮음.
  • 속도(Efficiency): 전송 과정에서 불필요한 클릭과 로그인을 배제하여 업무 효율성 극대화.
  • 경량화(Lightweight): 복잡한 설정 없이 실행 파일 하나로 구동되는 포터블 형태의 유틸리티.

5. 한계점 및 향후 개선 방향 (Todo)

현 버전의 한계: 동일 Wi-Fi 네트워크 내에서만 전송이 가능하며, 대용량 폴더 전송 기능은 미구현 상태입니다.

  • 외부망 전송을 위한 NAT 트래버스(Tunneling) 기술 도입 검토
  • 다중 파일 전송을 위한 ZIP 자동 압축 엔진 탑재
  • 전송 보안을 위한 1회용 인증키(OTP) 매칭 시스템 구축
